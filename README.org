* Usage 

** Build

The build process produces the following outputs:
- frontend files: ~tinydis.js~, ~tinydis.css~ and ~tindydis_bg.wasm~
- backend excutable binary: ~tinydis~

#+begin_src bash
  git clone https://github.com/cnglen/tinydis.git
  cd tinydis
  cargo install --locked cargo-leptos
  cargo leptos build --release && mv target/site/assets/tinydis.wasm  target/site/assets/tinydis_bg.wasm && rm -rf ./target/site/assets/*.ts
  cp -r ./target/site/assets YOUR_SITE_ROOT_PATH
#+end_src

** Start backend server

The backend server if responsible for:
- Reading/writing comments to the sqlite db
- Sending comments along with clickable reject/approve links to admin email for review

#+begin_src bash
  export TINYDIS_SMTP_USERNAME=your_email_to_send_review_message
  export TINYDIS_SMTP_PASSWORD=your_password_to_send_review_message   # an App password is prefered to regular parssword
  export TINYDIS_SMTP_HOST=your_smtp_host
  export TINYDIS_SMTP_PORT=your_smtp_port
  export TINYDIS_ADMIN_EMAIL=admin_email_to_receive_review_message
  export TINYDIS_ADMIN_LOCALE=en # Supported: en/zh-CN; default to en if not set
  export TINYDIS_ALLOWED_ORIGINS="*" # Allowed origins for backend server, e.g., "http://127.0.0.1:8080,http://127.0.0.1:9999" or "*" for testing

  ./PATH_OF_tinydis
#+end_src

** Import tinydis in frontend

To embed tindis in you HTML, the following attributes can be customized:
- ÔÇő=data-page-id=ÔÇő: defaults to the current page URL
- ÔÇő=data-server-url=ÔÇő: defaults to the HTTP *Origin* header
  
#+begin_src html
  <head>
    <!-- ... -->
    <link rel="stylesheet" href="/assets/tinydis.css" />
    <!-- ... -->
  </head>    

  <body>
    <!-- ... -->
    <div id="tinydis"></div>
    <!-- Optionally, customize data-page-id(for each page) and data-server-url -->
    <!-- <div id="tinydis" data-page-id="YOUR_PAGE_ID" data-server-url="TINYDIS_SERVER_ADDR"></div> -->
    <script type="module">
      import init from '/assets/tinydis.js';
      init();
    </script>
  </body>
#+end_src
